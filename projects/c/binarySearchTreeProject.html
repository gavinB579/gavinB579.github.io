C Binary Search Tree Showcase -

This project was probably the most difficult project I have done in C, and definitely the most difficult project I had to complete for my Computer Science 1 course. It incorported a lot of concepts, some of which I had not directly worked with before, even if I had seen similar concepts in my other projects. This project called for the use of dynamic memory allocation, pointers, a Quick Sort algorithm, and recursion to create a binary search tree that stores customer names and points. The user could add customers, subtract points from customers, delete customers, search how many points a customer has, and can count how many nodes in the BST are smaller than a specific customer.

A huge challenge to this program was the BST and the Quick Sort algorithm. I had a very tough time trying to implement the Quick Sort algorithm in a way that both functioned correctly and also ran in the correct time complexity. The bigger challenge was the binary search tree problem. To correctly implement it, I had to handle a lot of edge cases and reformating of the tree after certain operations (i.e. if a customer is deleted, the tree has to be reformatted to maintain the properties of a binary search tree). I had to use recursion for a lot of the operations on the tree, and I had to make sure that I was handling the base cases correctly. Additionally, it had to run in an acceptable time complexity, so I had to make sure that I was not doing any unnecessary operations on the tree. Overall, it was a very challenging project, but I learned a lot from it.
